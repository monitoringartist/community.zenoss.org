Zenoss 3.0 Release Candidate 1 (RC1) has been posted!!!

Posted by bbibeault Jul 9, 2010

It's finally here! The new Zenoss 3.0.0 release is right around the corner.  We've hit our first milestone of no outstanding P1 tickets (as of the time the build was generated), so we are labeling this artifact Zenoss 3.0.0 RC1.  What this means:

 

1) This build is a release candidate, not a release.  This is not the officially supported Zenoss 3.0.0 release.

2) This is a release candidate build; if no blocking issues are found, then this build could be promoted to a gold release build
3) This build is *the* build to test as quickly and thoroughly as you can - anything we can catch before release can go into the 3.0.0 gold release.

 

This build is unique in that it now involves the pre-upgrade zenpack, and new upgrade instructions (a first for Zenoss since I have been here).  These changes are around the move to the 3.0.0 framework, and I do not think will continue for future builds.

 

Location of the bits:

  •   Instructions for upgrading -  http://sourceforge.net/projects/zenoss/files/zenoss-3.0.rc/Before_Upgrading_To_3.0.rc.txt
  •   PreUpgrade30 ZenPack - http://sourceforge.net/projects/zenoss/files/zenoss-3.0.rc/ZenPacks.zenoss.PreUpgrade30-1.0-py2.4.egg
  •   Zenoss 3.0.0 RC1 bits - http://sourceforge.net/projects/zenoss/files/zenoss-3.0.rc/

 

Due to the time crunch around a release, we will not be holding a QA test day for this build.  However, as always, we welcome any and all feedback that you can provide.  Tickets into the Trac system are reviewed on a daily basis (if not hourly) by some of the testers.  Posts to the zenoss-testing forum on this site are monitored as they come in.  And, as always, there is the #zenoss channel on IRC.  Please offer all of the feedback that you can in the next couple of days!

Zenoss 3.0 Beta #1 released, QA Test Day 06.10.10

Posted by bbibeault Jun 8, 2010

It's here!  While folks have been able to take a look at http://public-demo.zenoss.com/ (login is admin/zenoss) for some time, and the four Zenoss 3.0 Alpha releases were downloaded by a large number of folks, I am proud today to announce the first 3.0 beta! Since the last alpha that was posted, we have fixed over 100 outstanding tickets, and gotten upgrades from previous Zenoss versions running.  Note that for beta #1 the only artifacts available are RPMs; expect the stack installers to be along for beta #2.

 

From the original alpha blog post - "Zenoss 3.0 includes a number of bug fixes,  and, most importantly, a completed redesigned user interface, as well  as Python 2.6 and Zope 2.12.  The new user interface features an updated  framework, new navigation layout, and redesigned pages."

 

The upgrade process from 2.5.x to 3.0 should follow the same process from 2.4 to 2.5 (I would use those instructions for references).  I would love to get any feedback in the Zenoss Testing forum about issues you saw with the upgrades, so we can try and get those fixes in to make beta #2 (ostensibly next week).

 

This  Thursday, June 10th, from 10am EST until 5pm EST, the Zenoss QA team  will be available for collecting feedback, answering any questions, and  testing any issues that are reported with the 3.0 beta.  The 3.0 Beta  can be reached from our "alpha" server, http://alpha.zenoss.com/2.5.70-beta/.   For a quick glance at the outstanding Stone Crab tickets, you can also  view the developer queue Trac report (#3) - http://dev.zenoss.org/trac/report.

 

For  those of you that wish to join us on IRC, we will be running this session on

Server:  irc.freenode.net (port 6667)

Channel:  #zenoss-testing

 

We'll also record a transcript of the day's conversations and links will be available from the Testing and IRC pages.

Zenoss 2.5.2 Beta RC - QA Test Day

Posted by bbibeault Jan 29, 2010

Thursday, February 4th will be the third Zenoss QA Test Day, this time covering our 2.5.2 release candidates.  For a quick look at what is fixed to the 2.5.2 release, you can view report #6 on Trac - http://dev.zenoss.org/trac/report.  A number of fixes around the Event Console have made it in, as well as hardening around ZenPack installs, network hierarchy nesting fixes, and some zenmib fixes.  Altogether, 85 are fixed in 2.5.2.  The primary goal of this Test Day is to catch any issues that might make the launch difficult.  Special focus will again be directed to the updated Event Console and the resulting fixes.

QA Test Day Retrospective

Posted by bbibeault Dec 11, 2009

I wanted to take some time and discuss the first Zenoss QA Test Day that took place yesterday (covering the Event Console).  An entire transcript of the chat can be found online at docs/DOC-4611.

 

First, I wanted to thank all of the attendees that joined #zenoss-testing and participated in the discussion.  For the Test Day, 13 tickets (http://dev.zenoss.org/trac/query?status=assigned&status=backlog&status=new&order=priority&col=id&col=summary&col=status&col=type&col=priority&col=milestone&col=component&keywords=~testday) were inserted around the Event Console and the events documentation in the Administration guide.  8 new test cases around the Event Console were also added.

 

If anyone has any feedback about the test day (what worked well, what could have worked better), please make sure to either comment on this blog post, or add to the zenoss-testing forum post on this topic.

 

Some ideas that came from the QA group on how we might do things differently in the future:

  • Use a collaborative screen-sharing tool (such as dim dim) to help reproduce tickets
  • Have a separate Test Day involving the alpha/beta vs. current functionality
  • Better adherence to (and a better defined) agenda for the test day
  • Repetitive tasks/operations, since the chat lasts for 8 hours, repeatedly revisit areas on a regular basis

 

I would love to hear what other ideas folks have, and want to thank everyone for going easy on us with this first chat!

Zenoss QA Test Day

Posted by bbibeault Dec 7, 2009

This Thursday will be the inaugural Zenoss QA Test Day.  After reading up on a few other groups that seem to be using this test day with significant success, Matt recommended it to the QA group, and it seemed like a great idea.  To see the actual thread describing the test day, you can go directly to it: thread/12200. 

 

Based on some of the original feedback that came in when 2.5.0 went live, I suspect that a number of community members have reproducible edge cases that have not yet been documented.  My goal for this first test day is to get the outstanding defects entered into the system, and to get test cases to cover these situations in the future, such that we do not regress in the future.

 

My stretch goal is to get some community members interacting with directly with Test Link.  In order for Zenoss QA to scale with new features and continue to increase regression coverage, we will need assistance from the community.  The benefits go both directions, though, and I hope that more people will realize this as we move forward.
