3.0-Incompatible Community ZenPacks

VERSION 7 
Created on: Jul 13, 2010 6:01 PM by Matt Ray - Last Modified:  Aug 13, 2010 6:19 PM by Matt Ray

With the upgrade to Python 2.6 and Zope 2.12 in the upcoming Zenoss 3.0 release, there are a number of incompatibilties that may have been introduced to existing ZenPacks.  While we have encouraged users to test their existing ZenPacks, not every ZenPack will continue to work with 3.0 without upgrading it.  We have documentation for authors looking to upgrade their ZenPacks: ZenPack Conversion Tasks for Zenoss 3.0

 

These were tested building with Ubuntu 32-bit and a script to automate installing them (attached).  The ZenPack authors have been notified, feel free to submit corrections were possible.  As other ZenPacks are found to have errors or are fixed, they will be added or removed were appropriate from the list.

 

Here is a list of known potentially incompatible ZenPacks:

 

ALL .ZIP ZenPacks should not be expected to work, they were deprecated with the 2.5 release but may be converted to .EGGs (Migrating ZIP ZenPacks to EGGs)

 

ZenPacks that import simplejson will have issues because it is now part of Python 2.6 as 'json':

  • Filtered Device Issues Portlet

 

The following ZenPacks fail to build from source:

  • Fortigate SNMP Monitor
  • OpenLayers Map Portlet
  • Tru64 Unix
  • Xen Virtual Machine Report

 

The following ZenPacks had errors installing:

  • NetWare 6.5
  • Sentry CDU Monitor

 

Other ZenPacks that have had errors found through their installation and testing are listed on the Community ZenPack list.  The source of the Python 2.6 ZenPacks is http://dev.zenoss.org/zenpacks/  Feel free to download and evaluate them and provide feedback whether or not they actually work.  This archive will be periodically refreshed with new updates.
