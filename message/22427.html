    New ZenPacks in the Community
    Zenossian Kells Kearney has upgraded the graphical MIB Browser ZenPack
    to properly uninstall and "spoonfeed zenmib when directories are
    found". This handy ZenPack provides a graphical MIB browser and adds
    the ability to install MIBs via http. Since this ZenPack fixes a
    broken uninstaller, you need to uninstall it to upgrade it.

    To uninstall the MIB Browser 1.04 ZenPack:
    Install the new 1.05 version, this will throw an error and appear to
    fail and still report as version 1.04
    Run the command "zenpack --remove=ZenPack.community.mib_utils"

    Then install the ZenPacks.community.mib_utils-1.05-py2.4.egg normally.

    MIB Browser: http://www.zenoss.com/community/projects/zenpacks/mib-browser

    Community ZenPack Repository page: trac-zenpacks/wiki/ZenPackscommunitymibutils

    Thanks Kells!

    New ZenPacks in the Community
    Zenoss Community member Paul Winkeler has generously donated his new
    Barracuda Spam Appliance ZenPack. The template contains performance
    graphs for the inbound queue and the number of:
    * allowed
    * tagged
    * quarantined
    * virus
    * bad recipient
    * spam
    messages received by the Barracuda. The ZenPack takes a novel
    approach for a datasource, using a Perl script to call wget and screen-
    scrape the web page with statistics on performance. Check it out if
    you're developing something similar, source is on the Community
    ZenPack Repository.

    Barracuda Spam Appliance: http://www.zenoss.com/community/projects/zenpacks/barracuda

    Community ZenPack Repository: trac-zenpacks/wiki/ZenPacksPBnJSolutionsBarracudaWatcher

    Thanks Paul!

    New ZenPacks in the Community
    Big thanks to Zenoss Community member Bob Killick for providing the
    new DragonWave AirPair ZenPack. This ZenPack provides basic
    monitoring of DragonWave AirPair series radios, under /Devices/Network/
    Wireless/AirPair. It was tested with the AirPair 100. Elements
    configured for monitoring are as follow:
    * Receive Signal Level - as measured in dbm
    * Radio Transmit Power - as measured in dbm
    * Modem Throughput - as measured in Mb/s

    The ZenPack also includes the required AirPair series MIBs.

    DragonWave AirPair: http://www.zenoss.com/community/projects/zenpacks/dragonwave/
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPacksSLSMCDragonWave

    New ZenPacks in the Community
    The VMware ESX Filesystems ZenPack has been updated to 2.2.1 and works
    on upgrades from the VMwareEsx.zip and .egg ZenPacks and on fresh
    installs. I tested it with a 2.3.3 VM against ESX. The name has been
    updated from "VMware ESX" to "VMware ESX Filesystems", since it really
    just adds the missing filesystems. Upgrading this ZenPack has been
    quite educational in what sort of "gotchas" there are for migrating
    from a .Zip style ZenPack to an .Egg. I've documented the necessary
    changes in the Community ZenPack Repository wiki: trac-zenpacks/wiki/MigratingZenPacks

    VMwareESX ZenPack: http://www.zenoss.com/community/projects/zenpacks/vmwareesx
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPacksvmwareVMwareEsx

    Thanks to Salmoneus and everyone else who raised the issue and to
    Wouter D'Haeseleer for the original ZenPack.

    44. Mar 11, 2009 4:26 PM (in response to Matt Ray)
    New ZenPacks in the Community
    Community member Eric Feldman has generously provided the new NetWare
    6.5 ZenPack. This ZenPack provides OS and Filesystem modeling and
    performance monitoring. It Provides:
    * A new device class = /Devices/Server/NetWare
    * Filesystem modeling collector plugin and a Filesystem template
    for performance monitoring, including display and calculation of
    purgeable disk space on NetWare volumes (done using SNMP and a shell
    script routine on the Zenoss server to calculate usedBlocks). The
    _ADMIN volume is dropped since it’s not a “real” volume.
    * OS Modeling (memory and swap size) collector plugins
    * OS and memory performance monitoring via a Device template
    o Processor utilization
    o Load averages (1 min/5 min/15 min)
    o Connections and peak use
    o Packet Receive Buffers (Current and Max)
    o Directory Cache Buffers (Current and Max)
    o Service Processess (Current and Max)
    o Read/Write and disk I/O activity
    o Logical and Physical Memory breakdown (based on Remote
    Manager reports)
    * Thresholds/alerts for:
    o Abends
    o Page Faulted Memory Spaces
    o Directory Services open/closed
    o Time synchronizaton status
    * IP Service monitor for NCP (port 524)
    * zLink to standard Novell Remote Manager port (8008) on the
    Status tab
    * All current Novell MIB files added to Zenoss for trap handling
    * Software inventory info for SP5 and SP7 added to Novell vendor
    listing

    NetWare 6.5: http://www.zenoss.com/community/projects/zenpacks/netware
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPackscommunityNetWare

    Thanks Eric!
