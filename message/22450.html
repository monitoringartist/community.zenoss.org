    New ZenPacks in the Community
    Thanks to Steven O'Donnell for providing his new Event Transforms
    Report ZenPack. This ZenPack consists of a report that shows all of
    the Event classes that have an event transform specified within your
    Zenoss installation. The larger the number of event transforms, the
    harder it is to keep track of them all, so this is found to be a
    valuable report. This report is sortable by the event class name,
    EventClassKey, transform, and number of events in that class. It is
    very similar to the All EventMappings report.

    Event Transforms Report ZenPack: http://www.zenoss.com/community/projects/zenpacks/transform-report
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPackssodonnelltTransformReport

    Thanks Steven!

    New ZenPacks in the Community
    Thanks to Zenoss Community member Steven O'Donnell for the new VMware
    Virtual Machines Report ZenPack. This report depends on the Community

    New ZenPacks in the Community
    Zenoss Community member Egor Puzanov has generously provided a new HP
    ProLiant Monitoring ZenPack, providing very thorough coverage. This
    includes support for the hardware components:
    * CPUs
    * Fans
    * Temprature sensors
    * Power supplies (limited support)
    * Memory Modules
    * Disk array hard disks
    Logical disks and controllers (cpqida.mib)
    * Fibre Channel hard disks
    * Logical disks
    * HBAs
    * Chassis and controllers (cpqfca.mib)
    * SCSI and SAS hard disks
    Logical disks and controllers (cpqscsi.mib)
    * IDE hard disks
    Logical disks and controllers (cpqide.mib)
    * NICs (cpqnic.mib)
    * iLO Boards (cpqsm2.mib)
    * Other PCI Devices

    Other supported monitoring functions:
    * Modeling
    * SNMP performance (hard disks & nics)
    * SNMP traps (set the current component status)

    HP ProLiant Monitoring ZenPack: http://www.zenoss.com/community/projects/zenpacks/hpmon
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPackscommunityHPMon

    Thanks Egor!

    55. Apr 19, 2009 5:27 PM (in response to Matt Ray)
    New ZenPacks in the Community
    Thanks to Eric J. Edgar for providing his new Point Threshold ZenPack.
    This ZenPack adds a new threshold type for determining when to trigger
    an event. It takes a single discrete value for non min/max-type
    thresholds.

    It was written because the author was trying to monitor a printer-
    queue status, which returned '3' for ready, '4' for off and '5' for
    printing. A min/max threshold would have given false alerts when
    printing. This point threshold solves that use-case.

    Point Threshold: http://www.zenoss.com/community/projects/zenpacks/pointthreshold
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPackscommunityPointThreshold

    Thanks Eric!

    New ZenPacks in the Community
    Zenoss Community member Tonino Greco has generously provided a new
    Printer Toner ZenPack. This Zenpack consists of a modeler which will
    model printers using SNMP, and determine their toner cartridge levels.
    The modeler then passes these values back to Zenoss. A new tab is
    added to the Device details for printers - called Printer Toner. Under
    this new tab, the details per cartridge may be obtained. Perf data is
    calculated as a percentage and alerts can be set of these values.

    Printer Toner ZenPack: http://www.zenoss.com/community/projects/zenpacks/printertoner
    Community ZenPack Repository: trac-zenpacks/wiki/ZenPackscommunityPrinterToner

    Thanks Tonino!
