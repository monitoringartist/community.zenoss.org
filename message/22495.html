    New ZenPacks in the Community
    Zenoss Community member Egor Puzanov has generously provided a new
    WMI Data Source ZenPack. This ZenPack creates a new zenperfwmi daemon
    for making Windows Management Interface calls via a WMI Data Source.
    This can be used to wrap WQL calls that had been previously made with
    wmic as Command data sources. The advantage for the WMI Data Source is
    that credentials should be properly handled and performance may be
    improved. This ZenPack can be used as the basis of other WMI
    monitoring ZenPacks (Windows and IIS6 are coming real soon).

    WMI Data Source: http://www.zenoss.com/community/projects/zenpacks/wmidatasource
    WMI Data Source Wiki: trac-zenpacks/wiki/ZenPackscommunityWMIDataSource

    Thanks Egor!

    New ZenPacks in the Community
    Thanks to Egor Puzanov for providing his new WMI Windows Performance
    ZenPack. This WMI-based ZenPack creates a new WMIDevice template for
    monitoring Windows performance via a WMI Data Source. This template is
    available under /Server/Windows and can be bound instead of the
    standard Device template. It uses WMI instead of SNMP, with the
    intention of replacing the need for SNMP-Informant.

    101. Aug 7, 2009 11:49 PM (in response to Matt Ray)
    New ZenPacks in the Community
    Zenoss Community member Ross Vandegrift has generously provided
    hosting for a new TCP Performance ZenPack. This ZenPack collects SNMP
    statistics on TCP from the standards-based TCP-MIB. This MIB should be
    well-supported on nearly any SNMP-capable device. This ZenPack
    provides a TCP-MIB device class under /Devices that you may bind to
    pull TCP performance data. This ZenPack can be used to provide
    alerting for systems that are under a large TCP connection load. The
    author's idea is to use this to determine when a system is under
    attack, compromised, or misconfigured and doing something stupid.

    TCP Performance: http://www.zenoss.com/community/projects/zenpacks/tcp-mib

    Thanks Ross!
