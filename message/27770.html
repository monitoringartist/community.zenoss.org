Config cycle and modeler cycle

If you use the left-hand menu and choose Collectors (which used to be called Monitors), it shows you various values including:
Config Cycle Interval (mins) 360
Modeler Cycle Interval (mins) 720

The 2.3 version of the Admin Guide, page 55, says that "By default, the system is setup to perform complete remodeling every 6 hours." - but it says this under the zenmodeler daemon.

My belief is that the zenmodeler daemon actually runs every 12 hours by default (the 720 shown above), not eveery 6 hours. Is this correct???

    RE: Config cycle and modeler cycle
    Chet dropped some knowledge on us:

    The default modeling cycle is 720 minutes (12 hours). The documentation is wrong. The "config cycle interval" is how often some of the daemons will reload their configuration unless it has been updated asynchronously in the interim. Really the config cycle interval is something that doesn't matter much anymore since everything is asynchronously updated. By changing the "Modeler Cycle Interval (mins)" in the collector's configuration you can change the default modeling interval.

    This is going to be captured in the 2.3.3 Admin Guide update.

    Thanks,
    Matt
