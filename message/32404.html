    important is to set the environment-variable SMIPATH.
    Assuming that ZenOss is installed in /usr/local/zenoss/

    export SMIPATH=/usr/local/zenoss/common/share/snmp/mibs:/usr/local/zenoss/common/share/mibs/iana:/usr/local/zenoss/common/share/mibs/ietf:/usr/local/zenoss/common/share/mibs/irtf:/usr/local/zenoss/common/share/mibs/site:/usr/local/zenoss/common/share/mibs/tubs


    set this in zenoss-profile (.profile or .bashrc in $ZENHOME) will prevent for execute this command every time after new login as zenoss

    RE: Trouble with installing a mib.
    Hello Folke,

    I think I have a solution...

    before you install the netcom-MIB-file, edit it and change the following line:

    Original:
    IMPORTS
     OBJECT-TYPE
          FROM SNMPv2-SMI-v1 <--
        TRAP-TYPE
                FROM RFC-1215
        DisplayString
                FROM RFC1213-MIB
        OBJECT-TYPE
                FROM RFC-1212
     MODULE-COMPLIANCE, OBJECT-GROUP
          FROM SNMPv2-CONF
     TEXTUAL-CONVENTION
          FROM SNMPv2-TC-v1 <--(delete "-v1")
     ;


    Change to:
    IMPORTS
     OBJECT-TYPE
          FROM SNMPv2-SMI <--(delete "-v1")
        TRAP-TYPE
                FROM RFC-1215
        DisplayString
                FROM RFC1213-MIB
        OBJECT-TYPE
                FROM RFC-1212
     MODULE-COMPLIANCE, OBJECT-GROUP
          FROM SNMPv2-CONF
     TEXTUAL-CONVENTION
          FROM SNMPv2-TC <--(delete "-v1")
     ;


    hope it helps

    regards
