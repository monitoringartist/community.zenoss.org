    8. Sep 7, 2009 10:38 AM (in response to Pim Rupert)
    workaround
    I see the exact same behavior -- every time the command runs, events get cleared and re-created (sending out notifications every time!)

    It looks like this is a regression caused by the fixes to tickets #5100 and #5469 (specifically, changeset [14772]).

    Here is a temporary fix until they come up with a patch:

    Edit /home/zenoss/Products/ZenRRD/zencommand.py line 593 (this should be a few lines below "def finished(self, cmdOrErr):"

    Change

    
cmd = cmdOrErr
self.sendCmdEvent(cmd, Clear, "Clear")
self.parseResults(cmd)


    to:

    
cmd = cmdOrErr
#self.sendCmdEvent(cmd, Clear, "Clear")
self.parseResults(cmd)


    ...and then restart zencommand (zencommand stop; zencommand start).

    For some reason zenoss is clearing the event before parsing it! This is obviously wrong... the parser should be the one solely responsible for parsing the result of the command and then issuing the clear or non-clear event based on the result.

    With the above change its working correctly again on my system (failure events accumulate without clearing, and then are properly cleared when the parser indicates they should be (i.e. 0 exit code for a nagios plugin command)). No need to live with all those spurious events on a holiday! --Kevin
