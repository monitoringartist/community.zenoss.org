    Repositories

    "mray" wrote:

     

    When you say there's no code repo, what specifically would you like?
    If I set up a project on SourceForge and pointed people towards that,
    would that work? I'd be more than happy to add a project and add
    anyone to it who wants to work on ZenPacks.


    Choosing a repository tools is the most controversial topic after editors and programming language. But there is one common consensus: Nobody is using SourceForge for software development any more. Although it is still heavily used for websites, mailinglists and software download.

    Which leads us into the more controversial area of actual choices:
    [*:b027a69167] https://launchpad.net - Which is of course my personal favorite (but I'm the guy who is asking for Ubuntu Beta packages)
    [*:b027a69167] http://developer.berlios.de - Great site that does a wonderful job, but isn't known very well outside Germany (which is a pity)
    [*:b027a69167] http://github.com - Fully Web 2.0 compliant, but you have to pay for it.
    [*:b027a69167] http://code.google.com - Which I'm sure does work pretty well, but their Terms of Service Statement does compete with the Microsoft EULA in length.
    [*:b027a69167] Setting up your own repository along the line of e.g. http://dev.laptop.org

    Personaly I would suggest to start hosting and developing the Zenoss Core ZenPacks in a repository system that allows commit access for external developers. That would not only make it easier to contribute to them, but also make it possible to develop your own ZenPack using the same system.
