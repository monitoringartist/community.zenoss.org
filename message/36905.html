    Solved error NoAttribute
    Okay, managed to resolve the zenpack cannot be removed error.

    Stop the zenoss: /etc/init.d/zenoss stop
    Start only the zeo: zeoctl start
    Go to Product folder within $ZENHOME: cd /usr/local/zenoss/Products
    Copy the problematic zenpack to /home/zenoss
    Remove the one inside Products folder

    (If anyone notice, within the Setting tab, the status will show that the zenpack is broken or missing, please restart Zope)

    Reinstall back the zenpack by running: zenpack --install <problematic zenpack>

    Run zendmd: dmd.Devices.reIndex(), commit()

    Run the zenmigrate --step=MakeZenPackManager

    (If anyone notice, before we run the zenpack --install, the zenmigrate --step=MakeZenPackManager will return error as shown above)

    The command zenmigrate --step=MakeZenPackManager will successfully executed.

    Run normal zenmigrate.

    Stop the zeo: zeoctl stop

    Start back the zenoss: /etc/init.d/zenoss start

    Checking within the Settings - > Daemon that all processes started

    Go to dashboard, refresh the screen and try to add the problematic zenpack back.

    In my case, the portlet has return to the usable state, again.
