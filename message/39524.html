    RE: WMI access on domain controller
    To avoid using a Domain Admin account for this I created a new Active Directory user with very little permissions. The reasoning behind this is to avoid any issues if the account were to ever get compromised since the password needs to be hardcoded into Zenoss so I can't expire it like I do with all my other AD user accounts.

    I then went to the WMI Control MMC of the systems that would be monitored using WMI, added the user account I created, and gave it the permissions it needed to access the Root namespace and everything under it.

    RE: WMI access on domain controller
    If you take the information in a few other posts about creating a non-Administrator user with all the access rights you need for full Zenoss monitoring (event log & services get tricky) you can create a Group Policy for it without too much trouble.

    The hardest part is giving access to the WMI namespace because there isn't a direct way of doing this via a Group Policy. See http://blogs.msdn.com/spatdsg/archive/2007/11/21/set-wmi-namespace-security-via-gpo-script.aspx for a method to do it.

    Re: Problem WMI user in a Domain

    Hi,

     

    I had the same problem,

     

    you need to add your domain user to the to the domain groups distributed com users and performance monitor users which is located in the builtin container, after that depending on how many domain controllers who have you need to modify the WMI control permission on each server, this can be done manually or via a little utility located here

     

    one last thing to do is give that user acess rights to this in the GPO under Computer Confiuration/Windows Settings/Security Settings/Local Policy/User Rights Assignment

     

    See http://technet.microsoft.com/en-us/library/cc738268%28WS.10%29.asp

     

    alternatively http://support.microsoft.com/kb/323076

     

    Finally for zenwin to work properly you need to give that user access to the service control manager on each server.

     

    http://support.microsoft.com/kb/907460

     

    Voila..

     

    Good luck, you're gonna need it.
