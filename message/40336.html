    Changing the test frequency at the datasource is correct. Any value should work, however, it needs to test at an interval compatible with your rrd-create statements.     Re: New IP-SLA Enumeration/Monitoring ZenPack
    I wanted to mention that the $timeout= value in each of the perl scripts is the amount of time the babysitter thread will wait for the query thread to finish. This is to prevent a query from hanging and piling up perl instances on the machine. If the query thread isnt done before the babysitter timeout is met, the babysitter thread will kill the query thread and then itself (even though it sounds like a news story :-D). This is especially important if you're using hundreds or thousands of SLA tests. Perl pile ups can eat up huge amounts of RAM and create insane process competition for CPU time. The next version of the SLA modeler is all in python; no more perl.
