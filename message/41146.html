    Your question is indeed quite vague. With something like Zenoss, there

    really isn't any meaning to maximum data. You need to know (and tell us

    if you want help) what you want to monitor. What do you want to know

    about the systems you are monitoring?

     

    For your modeling issue, that is the lack of OS information, you might

    want to look at the logs. On my test XP computer, I get:

    OS

    Tag #

     

    Serial #

     

    HW Make

    Microsoft

    HW Model

    .1.3.6.1.4.1.311.1.1.3.1.1

    Microsoft

    OS Version

    Windows 2000 Version 5.1

    Rack Slot

    0

    Name

    PC140

    Contact

     

    Location

     

    As you can see, this is basically from SNMP configuration, so I haven't

    entered TAG, SERIAL, Contact or location, so most of it is generic. I'd

    Zenoss server IP can pull, and to see if everything you want reported is

    Re: Status tab - how to fill information in?
    If it sets the model as an SNMP OID (a bunch of numbers with periods between them), then it's because it's not finding a matching model in the Products section.  Click on Products in the left hand menu and you'll see what I mean.  You may need to add products to the list by hand time to time.  You would select an appropriate section (such as Microsoft, or whatever), then add an entry, and specify the OID that it pulled down when you modeled the device.  You will also want to navigate to /Products/Unknown and remove the OID from that section afterwards as it gets added there automatically and will screw things up (since Zenoss might look at that entry before it sees the one that you added).
