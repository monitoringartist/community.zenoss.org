    Re: zenstatus issues after 2.4.5 -> 2.5 upgrade

    Then that's your problem.  If you navigate to the ZenPack page for the WMI Data Source ZenPack you'll see that I've added a warning at the top that states that the WMI Data Source ZenPack breaks 2.5 installs.  It needs to either be completely removed or upgraded to the newer version of the ZenPack (which is not actually available yet, only a BETA version of the new ZenPack has been posted).  I've tested an upgrade using the BETA version of the ZenPack and it worked just fine.  Unfortunately, now that you've upgraded to 2.5 with the WMI Data Source ZenPack installed you are completely out of luck.  I've tried bringing a 2.5 install with the WMI Data Source ZenPack installed back from the dead and I couldn't since the daemons wouldn't start, couldn't use the zenpack command to remove it, zendmd wouldn't start, etc...  Your install is totally trashed at this point and you are going to have to reinstall from scratch and restore from backups (which I hope you did).  If you're running on a VM and took a snapshot then even better, revert to the snapshot.

     

    This is why it is extremely important to test updates on a duplicate test server before actually performing updates on a production server.

     

    Regards,

     

    Ryan Matte
