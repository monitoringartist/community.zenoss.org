    Re: Help with Custom Map Zenpack

    An irritating feature about portlets is that they require an absolute path when you register them.

     

    In zendmd, what is the output of the following lines of code:

     

    for p in dmd.zport.ZenPortletManager.get_portlets():
        print p._getSourcePath(), p.check()

     

    If the MapPortlet.js is listed there, and its p.check is true, then let me know if you have any javascript errors in your browser's javascript console.

     

    If MapPortlet.js is not listed, try these lines of code:

     

    from Products.ZenModel.ZenossSecurity import *

    dmd.zport.ZenPortletManager.register_portlet(sourcepath="/usr/local/zenoss/zenoss/ZenPacks/ZenPacks.USU.Map/ZenPacks/USU/Map/skins/ZenPacks.USU.Map/MapPortlet.js", id='MapPortlet', title='Custom Map Portlet', permission=ZEN_COMMON)

     

    the __init__.py is supposed to do that for you, but only if you installed the zenpack from an egg, which you probably didn't do if you followed my install instructions. Hopefully ill get this cleaned up in and in egg form soon.
