    Re: Help with Custom Map Zenpack

    Wow... I am really glad I put that error check in there. I am assuming that test is a child location of the Location you are viewing.

     

    If you click on a device, it should send you to the page of that device. If you click on a submap, it should refresh the map screen. this is only possible if the submap also has the custom_map attribute, where the test map doesn't appear to have it.

     

    When you first load the zenpack, it attempts to modify every Location object to have the custom map. New classes however, won't have this attribute, or its associated tab. I don't think that the zope database stores the Location class, so I can't monkey patch it.

     

    As a compromise, I did add a method that checks the child locations for the attribute every time you load the map in the editor. That way, if the tab is missing, you should just be able to visit the map editing tab of the parent location to create it.

     

    This is one of the issues that I seriously need help with if anybody has any suggestions. Meanwhile, try out the attached egg.
