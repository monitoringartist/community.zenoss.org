    Re: Displaying an OID in Status tab

    First of all to push data to a field in the status tab you will need to make a custom modeler that will gather that for you.  Now the following procedure worked for me but if anyone has ways of making this better please feel free to chime in.

     

    So you will need to go to  /usr/local/zenoss/zenoss/Products/DataCollector/plugins/zenoss/snmp/

     

    Here you have a bunch of python scripts that are your collectors, you need to duplicate and rename one of those, you will be editing it to suit your needs.  Your version should look something like this: (I leave out the commented top of the script)

     

    __doc__ = """ChemistDeviceMap

     

    Gather info on Chemist's gear

    """

     

    from Products.DataCollector.plugins.CollectorPlugin import SnmpPlugin, GetMap

     

    class ChemistDeviceMap(SnmpPlugin)

         """Map mib element from Chemist's gear to comments or HW tag field

         """

     

         maptype = "ChemistDeviceMap"

     

         snmpGetMap = GetMap({

         '.1.3.6.1.4.1.x.x.x.x.x' : 'setHWTag',

         '.1.3.6.1.4.1.x.x.x.x.y' : 'comments'

         })

     

    def process(self, device, results, log):

         """collect snmp information from this device"""

         log.info('processing %s for device %s', self.name(), device.id)

         getdata, tabledata = results

         om = self.objectMap(getdata)

         return om

     

     

    Save this script as ChemistDeviceMap.py (or whatever).

     

    Now for this collector plugin to actually show up, you need to restart some component of Zenoss, now since I develop on a test machine I simply restart the whole Zenoss process but if you are in a production environment you might just want to restart the necessary daemons, I'm not sure which ones need to be restarted, however you might find that info in the forums.

     

    If you want this to look good, you will then navigate to the root of your device tree, http://xxx.xxx.xxx.xxx:8080/zport/dmd/Devices

     

    then you will choose: More -> Custom Schema

     

    You will add 2 custom schemas set the values as such:

     

    Label: cCustomCommentsHeader

    Name: cCustomCommentsHeader

    Type: string

    Default: Comments

     

    Label: cCustomHWTagHeader

    Name: cCustomHWTagHeader

    Type: string

    Default: Tag #

     

    Next, go to http://xxx.xxx.xxx.xxx:8080/zport/manage

     

    Click on portal_skins, then zenmodel, then deviceStatus

     

    Click the Customize button

     

    Replace the line:

     

    <dt class="first">Tag #</dt>


    
with


    

    
<dt class="first" tal:content="here/cCustomHWTagHeader"></dt>  and replace the line:

     

    <dt>Comments</dt>

with


    <dt tal:content="here/cCustomCommentsHeader"></dt>

     

    Now what we have done is create 2 variables, set their default values to what is normally shown as header text at the root of the device tree, and modified the skin to use these variable to replace the header text of the Comments and Tag # fields.  Since custom schema values are inherited throughout the device tree for all devices that we don't customize, the status page will look as it always has.  For the devices that we want to use these 2 fields for different data we will be able to customize the header text for the field.

     

    Now you need to create a sub-device class for your specific gear, so go to the root of the device (or wherever appropriate for the device type you are working on) tree and create a new device class, for example ChemistsUberGear, now navigate to that device class and go to:  More -> collector plugins

     

    There you want to add ChemistDeviceMap to the list (at the end of the list), this is what will overwrite the Tag # and Comments field with the OID values you specified in the ChemistDeviceMap.py collector script you made.

     

    Then go to: More -> custom schema

     

    Here you will set the values for cCustomCommentsHeader and cCustomHWTagHeader that you want displayed for this device class.

     

    So if the OID value you want to show in the comments field is a firmware version, put "Firmware", or any other string that describes the value you need to display.

     

    At this point, you are querying specific OID info from your device via your custom modeler, pushing that to the Comments or Tag # field, and we have hijacked the field header text via a variable so we can customize the header of these field per device class.

     

    Hope this helps

     

    Manuel
