Extending OSProcess

Was looking to get some feedback on extending OSProcess. I'll start with the background information so hopefully it makes a little more sense where I am coming from.

 

We run internally developed java applications. We will run multiple (anywhere from 5-10) instances of the application on a single host with each instance being identical except for the TCP port its listening on. I'd like to have a performance template setup to graph some of the very standard java metrics, such as HEAP usage. When we first setup Zenoss we simply created a copy of the template for each instance, update the revelant port info for JMX connections and called it day.

 

Well the day has come where we want to add 1 new metric across the board. Given we have multiple templates the work duplication is once again going to bite us in the (you know what). So it got me to thinking if I could list the applications as specific component, lets call it "MyApps". From there I could define a single template named "MyApps", using the JMX zenpack. I'm pretty sure in the template I use tales expression to pull the unique port out of the process instance (assuming of course I set the port as an attribute when I modeled it).

 

I'm pretty comfortable with writing the modeler to handler this, however my question is around how to best extend OSProcess, or if that is even the right thing to do. As I understand how relationships work. If I define a new component type, I would need to create a new device class. If I understand it correctly, this is required because of the way the bi-directional relationships are define in the object class files.

 

The goal would be to keep the device class as the standard "Device" that they are now, but enable support this new type of component, which leads me to wondering if extending OSProcess would do the trick?? I'd like to avoid hacking up the root device or OSProcess object class so I dont limit myself for future upgrades.

 

Am I thinking about this completely wrong, or will I be able to simply extend OSProcess, and my extra relationship and be off to the races?

 

Thanks
