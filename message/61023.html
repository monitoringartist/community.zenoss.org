Problems with LDAPserver and SSL-check templates

Hi,

 

yesterday I tried to add two new templates to a device: LDAPserver and SSL-certificate. The first one is a core ZenPack, the second one a community-made ZenPack.

 

At first the LDAPserver showed signs of doing some work: it gave an error because I hadn't configured it right yet. So it checked on the standard port and version 2. When I checked SSL and changed the port tot 636 and version to 3, it kept on checking the old (wrong) configuration. So I restarted Zenoss.

Now it won't check at all anymore. I cannot find anything in the logs about the LDAPserver or the SSL-check. Even when I put it on debug. Also, the values in the graphs remain "nan' and there is no RRD created. Also, I cannot change the "Cycle time" of the LDAPServer template, it remains empty.

 

I tried removing and readding the templates, stopping and starting Zenoss, but all without success. When I add another template, b_fping for example, ut works as expected. When I add the LDAPserver template to a device, I can see the following log-entry: INFO zen.zencommand: config: <Products.ZenRRD.zencommand.DeviceConfig instance at 0x5a13200>

But when I delete the b_fping-template, I can see te following log-entry: INFO zen.zencommand: Deleting command 192.168.123.53 from /usr/sbin/fping -c 3 -.... So here an whole command is added and deleted, which seems the way it should be.

 

When I try the SSL-check via the test-option manually and the LDAPserver command from the command prompt, they both work.

 

Anyone asuggestion as to what I'm (or Zenoss is) doieng wrong?

I'm using Zenoss 3.1.

    Re: Problems with LDAPserver and SSL-check templates

    This looks like a bug. The form to configure that LDAP datasource is saving

    a string instead of an integer and causing problems. As a quick fix you can

    edit the following file..

     

    /opt/zenoss/ZenPacks/ZenPacks.zenoss.LDAPMonitor-1.2.3-py2.6.egg/ZenPacks/zenoss/LDAPMonitor/datasources/LDAPMonitorDataSource.py

     

    And on line 100 change the "%d" to "%s" so that it becomes..

     

    parts.append('--ver%s' % self.ldapBindVersion)
