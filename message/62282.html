    Re: Zenoss 3.0 Compatible ZenPacks

    Hi John,

     

    1. if you have created simple ZenPack (some SNMP or Command data source based Performance templates and modeler plugins), than simply remove '-py2.4' or '-py2.6' part from .egg files names.

    2. if your ZenPack extends Zenoss objects model with new classes, datasources, zope page templates etc., you must add additional condition entry to configure.zcml file.

    <configure zcml:condition="installed Products.Zuul">

    this will prevent info.py and interfaces.py files from registering under zenoss 2.

     

    Egor

     

    PS: check any of my Zenpacks for more examples.

    mscomms wrote:

     

    Hi Matt, not sure if this is a compatability issue or just a newbie problem but

     

    community.HPUXmonitor-2.2-py2.4

    community.OpenSolaris-1.1-py2.4

    community.SunMibMonitor-1.0-py2.4

     

    all fail to load in 3.0.3 (this is a brand new appliance install)

    Hi,

     

    Without further inspecting i can tell you that zanpacks with "-py2.4" are for Zenoss 2.X and are automatically rejected by Zenoss 3.X.

     

    Zenpacks with "-py2.6" are compatibile with Zenoss 3.X.

     

    If you think this zenpack still *should* work with Zenos 3.x, remove the "-py2.4" part from Zenpack file name and it will be accepted by Zenoss 3.X, however this still doesn't mean it will function correctly.

     

    HTH

     

    Marko
