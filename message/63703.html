Consultant needed for Philadelphia company on new Zenoss Core 3.2.0 installation

I'm not sure if this is the proper forum for this but I hope it is.

 

My name is Shaun Walter. I work for a business consulting company in the Philadelphia PA area. We have recently installed a new Zenoss Core 3.2.0 server running on Oracle Enterprise Linux 5.6. We are replacing on older, very dodgy Zenoss 3.0.1 server. The new server is up and running Zenoss, I've been able to install several ZenPacks (Firestar.CiscoASA, WMIPerf_Windows, WMIPerf_Exchange2003, etc.) and add some additional plug-ins (check_oracle_health). We even have a good number of devices defined (about 25 servers, 10 network devices, 5 websites, 5 printers). Now we are either getting no alerts or too many alerts.

 

We are hoping to find someone to help us finish the configuration and tuning of our new Zenoss server. We have about 350 devices total to be monitored. I'd like to talk with a Zenoss consultant about our current status and how they can get us to a fully configured monitoring/alerting solution.

 

If anyone is available and qualified to help us, please contact me. We are eager to finish this and I'm hoping an experienced Zenoss consultant can provide the expertise we need to finish this in under a week. I can provide the information you need to price out a solution and provide a time frame for completion. This would include knowledge transfer so we can support the solution once it is complete.

 

If you're the person we are looking for, please reply. I'll be waiting.


Thanks.
