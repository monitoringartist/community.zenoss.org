    "esauve" wrote:

     

    What do I need to do to make french characters appear correctly? We have a few servers running the french version of Windows Server 2003 and although the events a properly picked up by Zenoss the french characters aren't displayed correctly. Thanks in advance.



    I've found the solution. Many thanks to bclz - his post gave me a clue.
    I did the following:

    1. Open Zope management console: http://<Zenoss server>:8080/manage
    2. Navigate to /zport/portal_skins/zenmodel/maintemplate-extras
    3. Edit the following portion of code: 
    
<tal:block metal:define-macro="head1">
    <meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1">
    Change charset to whatever you need, e.g. I changed to "charset=utf-8"
    4. Save changes and restart Zope

    At this point the main html page of Event Manager should have charset="utf-8" instead of defaults "iso-8859-1".

    To get correct characters in the ViewEventDetail-window, I did one more patch:
    5. In the template /zport/portal_skins/zenevents/viewEventFields I also changed iso-8859-1 to utf-8

    Now in my environment (Zenoss 2.4.2, RedHat 2.6.18 ) Zenoss Event Manager correctly displayes messages that comes from my Windows 2003 RUS servers! (Unfortunatly, this is partially true - see below sad )

    Added 2
    After a couple of experiments I come up with the following solution:

    6. In the directory $ZENHOME/python/ create file sitecustomize.py containing
    
[zenoss@zenoss python]$ more sitecustomize.py
import sys
sys.setdefaultencoding("UTF-8")


    7. Patch the file /usr/local/zenoss/zenoss/Products/ZenEvents/MySqlSendEvent.py. Change original string "value.encode('iso-8859-1')" to "value.encode('utf-8')".

    After such chunk of patches I can correctly see non-latin chars in the detailed view of the event (when pressing button in the right side of the row), but I still can't see non-latin chars in the "summary" column of the event table. Probably, zenoss aplies some additional conversion when rendering table.[/i]
