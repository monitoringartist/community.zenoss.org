    Re: Zenoss Core 3.0 / RHEL5 / LDAP Authentication


    I just managed to get LDAP auth. working against Windows AD.  Here is my best reconstruction of what I did.  Apologies for any gaps.

     

    Zenoss 3.0.2

    Red Hat Enterprise Linux Server release 5.5 (Tikanga)

     

    as user root:
    -----------------
    yum install openldap
    yum install python-ldap
    yum install gcc
    yum install openldap-devel
    yum install patch
    yum install openssl-devel
    yum install openldap-clients

     

    as user zenoss:
    --------------
    Note: I backed into this - ldap wasn't building and I didn't figure out
    that I had to hack the patch of constants.c until the end.

     

    You might start with python-ldap manual build (see below) and see how the Product.LDAP.... ones go after that.


    cd ~/
    mkdir build
    cd build
      download following:
        Products.LDAPMultiPlugins-1.12.tar.gz
        Products.LDAPUserFolder-2.18.tar.gz
        python-ldap-2.3.12.tar.gz

     

    cd ~/build
    tar zxvf Products.LDAPMultiPlugins-1.12.tar.gz
    cd Products.LDAPMultiPlugins-1.12
    python setup.py install
    (wait for it to fail)
    cd Products
    cp -a LDAPMultiPlugins/ /opt/zenoss/Products/

     

    cd ~/build
    tar zxvf Products.LDAPUserFolder-2.18.tar.gz
    cd Products.LDAPUserFolder-2.18
    python setup.py install
    (wait for it to fail)
    cd Products
    cp -a LDAPUserFolder/ /opt/zenoss/Products/

     

    cd ~/build
    tar zxvf python-ldap-2.3.12.tar.gz
    cd python-ldap-2.3.12
    cd Modules

     

    * --------------------------------
    * apply this patch to constants.c (found via google)
    --------------------------------
    (I did this manually by adding the 2 lines that start with '+'
    where they belong.)

     

    --- Modules/constants.c 7 May 2010 13:22:40 -0000       1.47
    +++ Modules/constants.c 11 Aug 2010 05:14:47 -0000
    @@ -181,7 +181,9 @@
    #ifdef LDAP_OPT_X_TLS_CRLCHECK
       /* only available if OpenSSL supports it => might cause backward
    compability problems */
            add_int(d,OPT_X_TLS_CRLCHECK);
    +#ifdef LDAP_OPT_X_TLS_CRLFILE
            add_int(d,OPT_X_TLS_CRLFILE);
    +#endif
            add_int(d,OPT_X_TLS_CRL_NONE);
            add_int(d,OPT_X_TLS_CRL_PEER);
            add_int(d,OPT_X_TLS_CRL_ALL);

     

    cd ..
    python setup.py install
    (should build cleanly)

     

    zopectl restart

     

    **check for errors in /opt/zenoss/log/event.log

     

    If all good follow normal instructions for adding LDAP provider (I copied mine from my
    Zenoss Core 2.5.0 config)

    Re: Zenoss Core 3.0 / RHEL5 / LDAP Authentication

    Yes, I just faced this problem last week when I evaluated what's involved with an upgrade from 2.5.4 to 3.0.2.

     

    You need a copy of the python-ldap code configured and compiled for python 2.6. I used the following method on CentOS5:

     

    • grab the python-ldap SRPM for my platform (CentOS5.4 - python-ldap-2.2.0-2.1.src.rpm) and install it
    • cd /usr/src/redhat/SPECS and build it with this command: rpmbuild -ba --define '__python /opt/zenoss/bin/python' python-ldap.spec
    • cd ../RPMS/x86_64 (presuming 64-bit install); mkdir temp; cd temp; rpm2cpio ../python-ldap-2.2.0-2.1.1.x86_64.rpm |cpio -dim
    • cd opt/zenoss/lib/python2.6/site-packages ; tar cf - * |tar xCf /opt/zenoss/lib/python2.6/site-packages -
    • clean up under /usr/src/redhat

     

    Let me know if this helps. If the rpm doesn't build for some reason, you need to make sure all prerequiste packages are installed, probably the same list

    that was posted in another reply.

     

    You could replace the existing python-ldap rpm with the one just compiled, but I'd strongly advise against it as it may break other software, and it's

    also not clear from the package name that it was compiled for a different prefix and python.

    Hi,

     

    I've just configure this successfuly on a CentOS 5.6 64bits Zenoss 3.2

     

    For those who need helps after install zenoss and followed the backup and installation steps in : http://community.zenoss.org/docs/DOC-2510

     

    I've installed python26 and python26-ldap from epel:

        

    yum install python26 python26-ldap

     

    then you'll need to copy the site-packages from /usr/lib[64]/python2.6/ to $ZENHOME/Lib/python2.6/site-packges:

     

    cp /usr/lib64/python2.6/site-packages/* $ZENHOME/Lib/python2.6/site-packages/

     

    and just restart zenoss (or zopectl only)

    If you have followed the instructions to use Active Directory or LDAP authentication with zenoss and you are having trouble seeing the Active Directory Multi Plugin after you have installed everything and restarted zenoss, this will help:

     

    Download http://pypi.python.org/pypi/python-ldap/2.3.13 (It is important to use this version)

     

    Then you need to build it with the zenoss/zope python version:

     

    /opt/zenoss/bin/python setup.py install

     

    Restart zenoss and then go to the zope management console and you will see Active Directory in the drop-down.
