    Re: SSH Configuration Port Change

    Dude:

     

    You'll want to delete the override for zCommandPort. This will cause the device to inherit the default form it's parent container. As for your model the reason it didn't return anything is that snmp monitoring is off for the device. You'll need to turn zSnmpIgnore to false and ensure the device can be walked by zenoss. Zenprocess monitors processes via snmp. As for the service monitoring that's handled sperately and will really only tell you if a service is up or down. Your best bet for monitoring SQL is to use Egors SQL datasource pack to acctually perform queries against mssql and graph / alert off of that.

     

    Best,

    --Shane
