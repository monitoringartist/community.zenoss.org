    Re: ssCpuIdle value showing zero

    Hello Bruce,

     

    I use the ssCpuRaw SNMP values and do calculations in my script to determine total CPU usage. The ssCpuRaw values are advantageous because Net-SNMP will report the total ticks per value, and this is not core-specific. So, you get an idea of your total CPU usage across all cores, regardless of count. If you want core-specific CPU tick data, this can be had, but it's not as simple (and I've always found that data rather worthless).

     

    Here's the command script that I use to get the CPU usage values and do the calculations:

     

     

    #!/usr/bin/perl -w

    use strict;

    #

    # Get Linux CPU statistics for ZenOSS/Nagios:

    #

     

    my $VERSION=$ARGV[0];

    my $COMM=$ARGV[1];

    my $HOST=$ARGV[2];

    my $sample_secs = 5;

     

    #

    # Define the OIDs:

    #

    my %OID = (

            User            => '1.3.6.1.4.1.2021.11.50.0',

            Nice            => '1.3.6.1.4.1.2021.11.51.0',

            System          => '1.3.6.1.4.1.2021.11.52.0',

            Idle            => '1.3.6.1.4.1.2021.11.53.0',

            Wait            => '1.3.6.1.4.1.2021.11.54.0',

            Kernel          => '1.3.6.1.4.1.2021.11.55.0',

            hwIRQ           => '1.3.6.1.4.1.2021.11.56.0',

            swIRQ           => '1.3.6.1.4.1.2021.11.61.0',

    );

     

    #

    # Get the CPU / Core count:

    #

    my $hrProcessorTable = '1.3.6.1.2.1.25.3.3.1.2';

    my $cpuCount = `snmpwalk -$VERSION -c$COMM -Ov $HOST $hrProcessorTable | wc -l`;

    chomp $cpuCount;

       

    #

    # Get the raw SNMP values:

    #

    my %delta1 = ();

    my %delta2 = ();

    my %result = ();

    my $all1 = 0;

    my $all2 = 0;

    while ( my($key, $val) = each(%OID) )

    {

            $delta1{$key} = `snmpget -$VERSION -c$COMM -Ov $HOST $val`;

            $delta1{$key} =~ s/.*: ([0-9]*)/$1/g;

            $all1 += $delta1{$key};

    }

    sleep $sample_secs;

    while ( my($key, $val) = each(%OID) )

    {

            $delta2{$key} = `snmpget -$VERSION -c$COMM -Ov $HOST $val`;

            $delta2{$key} =~ s/.*: ([0-9]*)/$1/g;

            $all2 += $delta2{$key};

    }

    while ( my($key, $val) = each(%OID) )

    {

            $result{$key} = $delta2{$key} - $delta1{$key};

    }

    my $all = $all2 - $all1;

    undef $all1; undef $all2;

    undef %delta1; undef %delta2;

     

     

    #

    # Group CPU stats into "standard" categories:

    #

    my %return = ();

    $return{'Kernel'} = $result{'System'} + $result{'hwIRQ'} + $result{'swIRQ'} + $result{'Kernel'} + $result{'Nice'};

    $return{'User'} = $result{'User'};

    $return{'Idle'} = $result{'Idle'};

    $return{'Wait'} = $result{'Wait'};

    $return{'TotalUsed'} = $return{'Kernel'} + $return{'User'} + $return{'Wait'};

    undef %result;

     

     

    #

    # Convert all values to % and return in Nagios format:

    #

    my $valPct = 0;

    my $returnString = "|Count=$cpuCount ";

    while ( my($key, $val) = each(%return) )

    {

            $valPct = sprintf("%.3f", $val / $all * 100);

            $returnString .= "$key=$valPct ";

    }

    print $returnString;

    Re: ssCpuIdle value showing zero

    The script should be used in your Data Source definition - set it up as a "Command" Data Source and use this script, making sure to pass the 3 required command variables - SNMP version, SNMP community, and hostname of server (can use TALES expression "${here/id}").

     

    However, I've since moved to using SSH for all Linux monitoring, and did develop an SSH-based Linux ZenPack that does some pretty awesome stuff. Net-SNMP has some major problems on RedHat, because they use an older version. And I've never gotten Net-SNMP to work flawlessly on any Unix OS, it's pretty much garbage, so SSH should be used if you're interested in quality statistics.
