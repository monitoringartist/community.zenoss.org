    Re: Help with Zenoss 4.2 -> "SNMP agent down - no response received" on localhost

    You are doing it correctly, but there are lots of options, as it's designed

    to be flexible to meet your needs. Below are some basic instructions

     

    snmpconf -g basic_setup

    Choose 'none' on the merge question, this way it will be a fresh file

    choose most of the default options #e.g. if you're not sure just hit enter

    Don't setup SNMP v3 or v2

    Setup v1 read-only

    give it a community name, example: dex-ro

    when done it should prompt you to save the file

    cp snmpd.conf /etc/snmp/snmpd.conf

    then restart snmpd
