/Status/Ping with critical severity

Hi everyone,
In Zenoss 4.2.0 Administration Documentation, the following was mentioned:

 

 

7.1.8.2.1. Example

The following example shows how to use the zensendevent script to simulate a ping down event:

zensendevent -d router1.example.com -s Critical -c /Status/Ping "Router down"

 

The thing is, when I issued the above command for one of my devices, an event has been created for that device with an event class /Status/Ping, and a severity=Critical. This however still displayed my device as being "UP" in the device status (With a green light), though I expect it to be red. I used to rely on this behavior in Zenoss 3.1.0 which is obviously not working in 4.2.0.

 

Is this a bug? Or did anything change in the new release?

 

Many thanks.

    Re: /Status/Ping with critical severity

    After checking some stuff here and there, I have realized that the following should be satisfied in order to see a "Down" device:
    1- The event class should be of type "/Status/Ping"
    2- The event severity shoud be "Critical"
    3- The IP address field for the device should be set
    4- For some weird reason, if the component field for your monitoring template is set to anything else than an empty field, the device status would still be up with the above criteria being met. You should keep the "Component" field empty, which maps to a "null" value later when the event is created

     

    I'm more than sure that without points 3 and 4 from the above list, Zenoss core 3.1.0 was able to consider a device as being down by simply satisfying points 1 and 2. Is this an intended change in Zenoss 4.2.0? Or is it a bug?
