    Re: RE: LDAP Authentication Failed password and Auth attempts

    Zenoss Enterprise (not core) provides a zenpack called LDAPAuthenticator that enables LDAP logins. We also provide a similar AD authenticator. The multiple logins was also an issue in the enterprise product that we fixed in a number of days, upon some customers reporting similar problems to our support team.

     

    The multiple login failures that occur for one failed password was an issue related to how our authentication system currently works. Basically what happens is after a failed password you are redirected to the "main" login screen and you fail the login at least one more time. We now make sure that only one authentication attempt is made to LDAP per user attempt.

     

    The fix exists as a patch for former vesions, and is included in the enterprise version of 2.5.2. Enterprise customers have access to this update.

     

    Just wanted to highlight some benefits of going enterprise with Zenoss :-)

    It is fixed for Enterprise Zenoss users >=2.5.2 . If you are on Enterprise 2.4.*, there is a new version of the ZenPack and a patch, installed with a single command, that support can give to Enterprise customers via the support portal.

     

    For core users,

    In /Products/LDAPUserFolder/LDAPDelegate.py

     

    apply this diff, or add in the + lines manually:

    +                if getattr(cached_user, 'invalid', None):
    +                    return None
    +
                     msg = 'getUserByAttr: "%s" cached in %s cache' % (
                         value, cache_type
                         )
    @@ -662,6 +670,15 @@

                 logger.debug('getUserByAttr: "%s=%s" not found' % (name, value))
    +
    +            # Cache the negative response to prevent locking accounts after
    +            # a single bad password attempt.
    +            if cache:
    +                user_obj = LDAPUser(value, value, pwd or 'undef',
    +                    [], [], user_dn, {}, {})
    +                user_obj.invalid = True
    +                self._cache(cache_type).set(value, user_obj)
    +
                 return None
