    Re: Graphs showing up as NAN

    The SNMP traps won't collect performance data.  Those will strictly be for notifications.

     

    I'm not familiar with Dell, but with HP gear you have to install their PSP packages on the OS (so Linux or Windows of the monitored device) and that installs an snmp agent that then talks to the hardware.  So you get the hardware configuration data through the OS not the management card.

     

    --Dennis

    Re: Graphs showing up as NAN

    Hi Andrew,

     

    I don't think you want to try to monitor server performance from the DRAC.  You'll probably get hardware alerts from it... but the DRAC has a different IP address from the server, doesn't it?  If you go to your graph's template and edit one of the data sources, you can test it against the addresses that are available on the server.  Find the one that responds correctly, and that's what you'll need to have set as the server's address. 

     

    Jay H.
