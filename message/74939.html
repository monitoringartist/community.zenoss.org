    At approx line 298, where the elif for TLDTYPE is for your ca domains put this in :

     

    elif [ "${TLDTYPE}" == "ca" ];
        then
                DOMAINDATE1=`cat ${WHOIS_TMP} | awk '/Expiry date:/ {print $NF}'`
                DOMAINDATE=`date -d"$DOMAINDATE1" +"%d-%b-%Y"`

    I think the biggest issue was the function call I used.  Normally I use the bash date command to convert times to epoch.  In the case of this ZenPack however I used a function call to do the math and conversions.  The problem is the date must be in an exact format.  the bash date command is much more flexable and will take the return values for the expiration date in a lot of different formats.  That was one of the things I was going to work on with the next version.  If you have a dev with the time, you might have him focus on that, since it will probably be the biggest blocker to making the script flexable enough to work with any standard whois server.  I also think that at that time it will not be as necessary to specify specific whois servers any longer.
