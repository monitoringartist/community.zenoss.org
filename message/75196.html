    Re: Can zenoss handle IBM Storwize V7000?

    I'm not familar with the IBM STOREWIZE V7000, but the answer to your question is "more than likely". If the device supports SNMP, than it would be an "almost certainly".  It really depends on what you mean by "handle". Without doing anything Zenoss will be able recieve SNMP traps. You would probably want to load the MIBS to have the traps translated.

     

    If your looking to have Zenoss collect historical performance data, you might need to create your own SNMP based monitoring template using the OIDs the device exposes.

    Re: Can zenoss handle IBM Storwize V7000?

    Hi handsy,

     

    it seems like IBM STOREWIZE V7000 supported SMI-S monitoring (search for IBM STOREWIZE V7000 CIM agent).

    So you can try to monitor your storage with https://github.com/epuzanov/ZenPacks.community.CIMMon ZenPack.

     

    1. set zCIMConnectionString and zCIMHWConnectionString zProperties on /Storage/SMI-S device class:

     

    'pywbemdb',scheme='https',host='ip_arrderss_of_server_with cim_agent_installed',port=5989,user='${here/zWinUser}',password='${here/zWinPassword}',namespace='root/ibm'

     

    2. set zWinUser and zWinPassword zProperties

    3. put your storage in  /Storage/SMI-S device class

    4. remodel it
