    Re: Error Monitoring Virtual-Machines after upgrading 2.4.5 to 2.5

    Hello Matt Ray,

     

    thanks for the great work. Well done ...

     

    The "Virtual Machines"-Tab and all other Sites are working now. There is no installation error. But there are errors in the template, that would be installed with the ZenPack as described here

     

    http://community.zenoss.org/message/28824#28824

     

    first of all the path to the "check_esx2" in the command-line of the data-source "CPU_Utilization" is not correct and there is missing a "|" in the command-line:

     

    it depends on the linux-distribution ...

     

    generally after .../zenoss/... it must be changed from

     

    .../zenoss/ZenPacks.vmware.VirtualMachines/...

     

    to

     

    $ZENHOME/ZenPacks/ZenPacks.vmware.VirtualMachines-1.2.1-py2.4.egg/...

     

    I don't know if there is a way to make this code-line generally for all distributions and all following ZenPack-Versions.

     

    And the last sed-command must be changed from

     

    ...|sed -e 's/.*CPU\susage\sis\s/cpuUtil=/;s#%##;s/on.*(/cpuSec=/;s#s##;s/average.*$$//'

     

    to

     

    ...|sed -e 's/.*CPU\susage\sis\s/|cpuUtil=/;s#%##;s/on.*(/cpuSec=/;s#s##;s/average.*$$//'

     

    And last but not least the Names of the Virtual Machines may not contain "Spaces" because otherwise the check_esx2 produces on all polls the result:


    No aved state for Root-DC 2 CPU time yet - wait for next poll.

     

    without this tweaking there woudn't draw a CPU-Utilization-Performance-Graph.

     

    Hope this help someone

     

    Regards

     

    dbuck

    Hello Matt Ray,

    Install it over the old ZenPack stops all Performance-Graphs for the Virtual machines.

     

    Steps to solve this (!!!with loosing all rrd-Graphs for the Virtual Machines!!!):

     

    As User zenoss

     

    1. uninstall "ZenPacks.sodonnell.VMReport" (if installed, because based on VitualMachines-Zenpack)

    2. uninstall "ZenPacks.vmware.VirtualMachines"

    3. stop Zenoss with "zenoss stop"

    4. delete all Directories under $ZENHOME/perf/Devices/ESX-Server/virtualmachines/

         (repeat this Step for all ESX-Server under $ZENHOME/perf/Devices/, where "ESX-Server" must be replaced by the name of your Esx-Server)

    5. start Zope with "zeoctl start"

    6. reindex Zope-Database with:

         zendmd

         >>reindex()

         >>commit()

         CTRL+D

    7. start Zenoss with "zenoss start"

    8. install the two ZenPacks from Step 1 and 2

    9. stop zenoss as in Step 3., repeat Step 5 to 7

     

    After some Time all Performance-Graphs of the Virtual-Machines starts Graphing

     

    I know this is the sledgehammer-method!
