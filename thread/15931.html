Rogue IP: Already assigned but can't be deleted!?

Hello folks,

 

I appear to have a rogue IP address in my Zope database adn it's causing me a bit of trouble.

 

When I model the server in question I get an "IP Address is already assigned" error. When I try to set the IP in zendmd, I get the same:

 

>>> d = dmd.Devices.findDevice('myserver.mydomain.com')

>>> d.setManageIp('10.66.128.53')
1
2011-04-21 14:26:08 WARNING zen.Device The IP address 10.66.128.53 is already assigned
'The IP address 10.66.128.53 is already assigned'

 

However, if I trawl through all IPs on all interfaces on all devices looking for IPs like 10.66.128.* the IP in question appears to not be allocated:

 

>>> for d in dmd.Devices.getSubDevices():
...     for iface in d.os.interfaces():
...         ip = iface.getIpAddress()
...         if ip == None:
...             continue
...         else:
...             if re.compile('10\.66\.128.*').search(ip):
...                 print d.id, iface.id, ip
...
************.********.com VMware Accelerated AMD PCNet Adapter - Packet Scheduler Miniport 10.66.128.62/24
************.********.com HP Network Team _1 10.66.128.51/24
************.********.com HP Network Team _1 10.66.128.38/24
************.********.com HP NC373i Multifunction Gigabit Server Adapter _2 10.66.128.35/24
************.********.com HP Network Team _1 10.66.128.44/24
************.********.com HP NC373i Multifunction Gigabit Server Adapter 10.66.128.31/24
************.********.com HP Network Team _1 10.66.128.57/24
************.********.com HP Network Team _1 10.66.128.43/24

 

So the IP is somewhere in the zope database, but apparently not tied to an interface on any of my devices. To confirm this, I had a look at the zope web interface:

 

 

However if I select the IP in question and click "delete", Zenoss throws an error and the IP remains.

 

Does anybody have any suggestions as to how I might get rid of this rogue IP address? I'm all out of ideas...

 

Thanks and Happy Easter,

 

James

    HI...

     

    I had this same problem before.

     

    This is what worked for me.....

     

    Browse to Advanced->Collectors->localhost. Then check to see if the device is listed in the "Devices" list.

    If it is...check the box next to it...then select "Delete devices" from the action menu (the gear next to "Devices".

     

    Tec
